Obligatorisk oppgåve 5 - 6121 Bildebehandling H2018

Arbeidet skal leverast som ein lab-rapport seinast mandag 24. september. Last opp pdf i Canvas. To til fire studentar på kvar gruppe, unntak frå dette berre etter skriftleg (epost-) søknad. Pass på at framsida klart fortel kva dette er (oblig 5 i 6121 hausten 2018) og kva forfattarane heiter.

  1. Lag ein plugg som implementerer histogramnormalisering, histogramspesifikasjon basert på at du skal oppnå eit normalfordelt (Gauss-fordelt) histogram. (Dette er altså ikkje det same som normalisert histogram!) Middelverdi og standardavvik skal vere parametre som brukar må gi i eit dialogvindauge, med 127 og 50 som default-verdiar.

    Leit sjøl opp formelen for normalfordeling, legg verdiane inn i ein tabell (histogram). Korleis dette er skalert, er ikkje viktig. Fra histogrammet reknar du på vanleg vis ut det kumulative histogrammet. Kva for skalering histogrammet hadde, vil styre kva verdiar dette kumulative histgrammet vil ha. Skalér slik at største verdi er 255, dvs. del på H[255] og multiplisér med 255. Dette nyttast som PR, jmf likning (4.20) i boka. Eit moment som står att, er å finne PR-1 basert på PR. Prøv om denne fungerer:

        public int[] inversFunksjon(int[] P){
      final int K = 256; //antal gråtoner
      int[] Pm1 = new int[K];
      for (int i=0; i<K; i++) {
        int p=P[i];
        Pm1[p] = i;
      }
      for (int i=1; i<K; i++)
        if (Pm1[i] == 0) Pm1[i] = Pm1[i-1];
      return Pm1;
    }

  2. Test denne pluggen på minst to bilde: vis bilde før og etter, vis histogram før og etter, og vis kumulativt histogram før og etter.

  3. Ved hjelp av ferdig funksjonalitet i ImageJ skal du lage demobilde som viser flest mogeleg måtar å kombinere to bilde - aritmetisk og logisk. Forklar i rapporten kva for operasjon som er nytta i dei ulike bilda.
